# How to Use `new.md`

This repository uses `new.md` as the **authoritative protocol** for initializing any new project.

## For Humans
1. Start a new project by telling the agent to follow `new.md`
2. Answer intake questions honestly and explicitly
3. Treat all approvals as intentional design decisions

## For the Dev Agent
- Follow `new.md` top-to-bottom
- Do not skip phases
- Do not implement without approval
- Track assumptions explicitly
- Treat documentation as a first-class artifact

If any step in `new.md` cannot be completed due to missing information:
- Stop
- Ask the human
- Document the open question
